## STEP 2 — ADD THE SERVER ROUTE

In `server.js` (or `index.js`, wherever Express routes are defined), add this route. It must be added **after** your session middleware is initialized so the session cookie is available:

```javascript
// NBS Proposal Log — serves the pre-built HTML module
app.get('/tools/proposal-log', (req, res) => {
  res.sendFile(path.join(__dirname, 'public', 'tools', 'proposal-log.html'));
});
```

Make sure `path` is already imported at the top of the file. If not, add:

```javascript
const path = require('path');
```

## STEP 4 — AUTH CONNECTION (READ CAREFULLY — NO CODE CHANGES NEEDED ON THE SERVER)

The proposal log HTML file already contains all auth logic. It calls your existing `GET /api/auth/me` endpoint on page load using:

```javascript
fetch('/api/auth/me', { credentials: 'include' })
```

The session cookie (`connect.sid`) is sent automatically because the proposal log is served from the same domain. **No changes are needed to your auth endpoint or session middleware.**

The endpoint must return this JSON shape (which it already does):

```json
{
  "user": {
    "id": 1,
    "email": "user@example.com",
    "role": "admin",
    "displayName": "Haley",
    "username": "hkkruse"
  }
}
```

If the user is not authenticated, it returns `401` — the proposal log will show a sign-in gate automatically.

**Role behavior (already built into the HTML — do not modify):**

- `role: "admin"` → Full access to everything
- `role: "user"` → Standard access only (see full breakdown below)

### IMPORT EXCEL MODAL (admin only)

Two-step modal, max-width 920px:

**Step 1 — Upload:**

- Large dashed dropzone area — supports drag-and-drop or click-to-browse
- Accepts `.xlsx` and `.xls` files only
- “Expected Columns” hint section shows: Project Name, Bid Date, Estimator, Proposal Amount, Swinerton Builders Contact, Project Location, Scope Start Date, Scope End Date, Status

**Step 2 — Preview (shown after file is parsed):**

- Summary row showing: Total / Ready (green) / Region? (orange) / Dupes (red) counts
- Scrollable preview table with columns: checkbox, row #, flag, project name, estimate #, bid date, estimator, amount, status, region, raw location, GC contact, start, end
- Each row flagged as: OK (green), REGION? (orange — location string couldn’t be matched to known region), or DUPE (red — project name matches existing entry)
- Duplicate rows are unchecked by default
- Header checkbox toggles all rows
- Footer shows “N of M rows selected” count + Cancel + “⬆ Import Selected” buttons

**Import logic:**

- Auto-generates estimate numbers in format `RB-YY001`, `RB-YY002` etc. where YY = 2-digit year from bid date, incrementing from the highest existing number for that year
- Fuzzy column header matching (case-insensitive partial match) for: project name, bid date, estimator, proposal amount, swinerton contact, location/region, scope start, scope end, status
- Date parsing handles: MM/DD/YY, MM/DD/YYYY, Excel serial numbers
- Scope date parsing handles: MM/YY format, ISO dates
- Dollar parsing strips $, commas, spaces
- Status mapping: “pending/submitted” → Submitted, “bidding/estimating/in progress” → Estimating, “revising/revision” → Revising, “lost/loss” → Lost, “win/won/awarded” → Awarded
- Auto-promotes Estimating → Submitted if dollar amount > 0
- Region fuzzy matching against the full 25-region list (exact match, prefix match, airport code match)
