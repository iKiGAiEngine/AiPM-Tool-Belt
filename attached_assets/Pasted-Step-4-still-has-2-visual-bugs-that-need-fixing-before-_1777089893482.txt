Bug 2: Panel does not fully hide when closed.

User reports: “If it’s collapsed, if I move the screen to the right, I can see it trying to hide on that side of the screen.”

The slide-out animation moves the panel off the viewport but the panel still exists in the layout, causing horizontal scroll/overflow. The user can scroll right and see the closed panel sitting just off-screen.

Required fix:

1. After the slide-out animation completes, the panel must be fully removed from the visual layout (use display: none, visibility: hidden, or unmount the panel from the DOM)
1. The body and main page container must have overflow-x: hidden to prevent any phantom horizontal scrolling
1. Verify on mobile that scrolling right does NOT reveal the closed panel

Recommended approach: use a state like isAnimatingOut - when user clicks close, run the slide animation, then on animation end set the panel to display: none (or unmount it). When user reopens, mount it first, then run the slide-in animation.
