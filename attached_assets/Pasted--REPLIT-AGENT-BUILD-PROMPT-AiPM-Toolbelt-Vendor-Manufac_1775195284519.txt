## WHAT TO BUILD

Build a **Vendor / Manufacturer Database** web application — a module within the **AiPM Toolbelt**, a construction technology platform for commercial specialty contractors. This is an internal tool for **National Building Specialties (NBS)**, a commercial construction company based in Riverside, CA that installs Division 10 specialties (toilet accessories, partitions, lockers, etc.).

The app has **two tabbed views** within one module:

1. **Vendors tab** — The daily-use view for Project Managers and Estimators. Vendor profiles with contacts, products, pricing, lead times, and a minimal tax summary.
1. **Certificate Tracker tab** — An accounting/admin view for managing resale certificate workflows (send → confirm → track expiration). Not needed day-to-day by PMs.

The app must support **Excel upload** of NBS’s existing manufacturer list to bulk-populate vendor and contact records on first use.

-----

## TECH STACK

- **Frontend:** React with Vite, styled with Tailwind CSS
- **Backend:** Express / Node.js REST API
- **Database:** Replit PostgreSQL (Neon) — use Replit’s built-in Postgres
- **Auth:** Simple email/password with bcrypt + JWT (no third-party auth providers)
- **File Storage:** Local filesystem `/uploads` directory for uploaded documents (W-9s, certs, price sheets)
- **File Parsing:** Use `xlsx` npm package (SheetJS) to parse uploaded Excel files

## DATABASE SCHEMA

Create these tables in PostgreSQL. Run migrations on app startup.

```sql
-- Users & Auth
CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  display_name VARCHAR(100),
  role VARCHAR(20) DEFAULT 'viewer' CHECK (role IN ('admin', 'pm', 'viewer')),
  created_at TIMESTAMP DEFAULT NOW(),
  last_login TIMESTAMP
);

-- Vendors (master record)
CREATE TABLE IF NOT EXISTS vendors (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,
  category VARCHAR(100),
  website VARCHAR(500),
  notes TEXT,
  tags TEXT[],
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Contacts (many per vendor)
CREATE TABLE IF NOT EXISTS contacts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE,
  name VARCHAR(255),
  role VARCHAR(100),
  email VARCHAR(255),
  phone VARCHAR(50),
  territory VARCHAR(255),
  is_primary BOOLEAN DEFAULT FALSE,
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Products (many per vendor)
CREATE TABLE IF NOT EXISTS products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE,
  model VARCHAR(255),
  description TEXT,
  csi_code VARCHAR(50),
  list_price DECIMAL(10,2),
  unit VARCHAR(20),
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Pricing (one per vendor)
CREATE TABLE IF NOT EXISTS pricing (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE UNIQUE,
  discount_tier VARCHAR(255),
  payment_terms VARCHAR(255),
  notes TEXT,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Logistics (one per vendor)
CREATE TABLE IF NOT EXISTS logistics (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE UNIQUE,
  avg_lead_time_days INTEGER,
  ships_from VARCHAR(255),
  freight_notes TEXT,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Tax Info (one per vendor)
CREATE TABLE IF NOT EXISTS tax_info (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE UNIQUE,
  ein VARCHAR(20),
  w9_on_file BOOLEAN DEFAULT FALSE,
  w9_received_date DATE,
  is_1099_eligible BOOLEAN DEFAULT FALSE,
  tax_exempt BOOLEAN DEFAULT FALSE,
  exemption_type VARCHAR(100),
  exemption_cert_number VARCHAR(100),
  nexus_states TEXT[],
  tax_notes TEXT,
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Resale Certificates (many per vendor, one per state)
CREATE TABLE IF NOT EXISTS resale_certificates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE,
  state CHAR(2) NOT NULL,
  cert_type VARCHAR(20) DEFAULT 'Resale',
  cert_number VARCHAR(100),
  issue_date DATE,
  expiration_date DATE,
  sent BOOLEAN DEFAULT FALSE,
  date_sent DATE,
  contact_sent_to VARCHAR(255),
  vendor_confirmed BOOLEAN DEFAULT FALSE,
  confirmation_date DATE,
  blanket BOOLEAN DEFAULT FALSE,
  project_name VARCHAR(255),
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Files (many per vendor)
CREATE TABLE IF NOT EXISTS files (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vendor_id UUID REFERENCES vendors(id) ON DELETE CASCADE,
  file_type VARCHAR(50),
  original_name VARCHAR(500),
  stored_path VARCHAR(1000),
  mime_type VARCHAR(100),
  size_bytes INTEGER,
  uploaded_by UUID REFERENCES users(id),
  uploaded_at TIMESTAMP DEFAULT NOW(),
  notes VARCHAR(500)
);

-- Seed a default admin user (password: "admin123" — change on first login)
-- Hash this password with bcrypt in the migration script
INSERT INTO users (email, password_hash, display_name, role)
VALUES ('admin@nbs.com', '$BCRYPT_HASH_OF_admin123', 'Admin', 'admin')
ON CONFLICT (email) DO NOTHING;
```

## EXCEL UPLOAD FEATURE (CRITICAL)

NBS has an existing manufacturer list in Excel format. The upload feature must parse this file and populate the database. Here is the exact structure of the Excel file:

### Excel Column Mapping

The spreadsheet has these columns (with a header row and a “Scope/Trade” sub-header row):

|Excel Column                                |Maps To                                                                                                                                                                                                                                                                                           |
|--------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|**Scope/ Trade**                            |`vendors.category` — values like “Toilet Accessories”, “Partitions”, etc. This is a section header row, not per-row. Apply the most recent Scope/Trade value to all rows below it until the next Scope/Trade row.                                                                                 |
|**Manufacturer**                            |`vendors.name` — e.g. “Bobrick”, “Bradley”, “ASI”. Multiple rows may reference the same manufacturer.                                                                                                                                                                                             |
|**Manufacturer Representative/ Distributor**|Create as a **contact** record with `role = "Manufacturer Rep / Distributor"` and `contacts.name` = this value (e.g. “Pacific Building Specialties”, “Total Restroom”, “Fastenal”)                                                                                                                |
|**Contact Name**                            |`contacts.name` — the actual person name (e.g. “Michael Clark”, “Ian Blair”). If a Rep/Distributor is also listed, create TWO contacts: one for the rep company and one for the person. Or better: set `contacts.name` to the person name and `contacts.role` to the rep/distributor company name.|
|**Email Contact**                           |`contacts.email` — may contain multiple emails separated by semicolons. If multiple, create multiple contact records or store the first as primary.                                                                                                                                               |

### Upload Processing Logic

```
1. Parse the Excel file using SheetJS (xlsx npm package)
2. Read the "Estimating" sheet (first sheet)
3. Skip the title row ("ESTIMATING") and header row
4. Track the current "Scope/Trade" — when a row has a value in column A but nothing in columns B-E, it's a scope header row
5. For each data row:
   a. Check if a vendor with this name already exists (case-insensitive match)
   b. If not, create vendor with name and category (from current scope)
   c. If the vendor already exists, skip vendor creation but still process contacts
   d. Create contact record(s):
      - name: Contact Name column value
      - role: Manufacturer Representative/Distributor column value
      - email: Email Contact column value (handle semicolons)
      - is_primary: true for the first contact per vendor
6. Return summary: { vendorsCreated, vendorsSkipped, contactsCreated }
```

### Upload UI

- In the Vendors tab header, add an “Upload Excel” button next to Import/Export
- Clicking opens a modal with:
  - Drag-and-drop zone or file picker (.xlsx, .xls only)
  - Brief instruction: “Upload your NBS Manufacturer List (.xlsx). The Estimating sheet will be parsed to create vendor and contact records.”
  - After upload, show a summary: “Created X vendors, Y contacts. Skipped Z existing vendors.”
  - Option to download a template file that matches the expected format

## CERTIFICATE TRACKING STATUS LOGIC

Compute the tracking status for each cert in the application layer (not stored in DB):

```javascript
function getCertTrackingStatus(cert) {
  // Check expiration first (overrides other statuses)
  if (cert.expiration_date) {
    const daysUntil = daysDifference(cert.expiration_date, today);
    if (daysUntil < 0) return "expired";
    if (daysUntil <= 90) return "expiring";
  }
  // Then check workflow state
  if (!cert.sent) return "not_sent";
  if (cert.vendor_confirmed) return "confirmed";
  return "sent";
}
```

## IMPORTANT IMPLEMENTATION NOTES

1. **GET /api/vendors/:id should JOIN all related tables** — return vendor + contacts + products + pricing + logistics + tax_info + resale_certificates + files in a single response. The frontend expects a fully hydrated vendor object.
1. **Search should be full-text across multiple tables** — when searching “Michael Clark” it should find the vendor that has that contact. Use PostgreSQL `ILIKE` across vendors.name, contacts.name, contacts.email, products.model, products.description, and any array element in vendors.tags.
1. **The Excel upload is the most important feature for first use** — if this doesn’t work smoothly, the tool is useless because nobody will re-enter 50+ vendors by hand. Test with the column layout described above.
1. **Backward compatibility for existing data** — the cert tracker fields (sent, date_sent, vendor_confirmed, etc.) should all default to false/null so existing cert records still work.
1. **Tab navigation state** — switching between Vendors and Certificate Tracker should preserve state within each tab (scroll position, open modals, etc.) but navigating to a vendor detail and back should return to the list.
1. **Single source of truth for cert data** — the Certificate Tracker tab and the vendor detail’s Tax & Compliance section both read/write the same `resale_certificates` table. Edits in one place must reflect in the other.
1. **Toast notifications** — show on every save, delete, upload, and import action. Auto-dismiss after 3 seconds. Position: fixed bottom-right.
1. **The app will eventually connect to other AiPM Toolbelt modules** — structure the codebase so vendor data can be referenced by future tools (submittal builder, procurement log, estimate-to-cart). The vendor API should be clean and well-documented.

## WHAT SUCCESS LOOKS LIKE

When the build is complete, a user should be able to:

1. Log in with admin@nbs.com / admin123
1. Upload the NBS Manufacturer List Excel file and see 30+ vendors auto-populated with contacts
1. Click into any vendor and see all their contacts pre-filled
1. Add products, pricing, and logistics info to vendors
1. Switch to the Certificate Tracker tab and see the dashboard showing “X vendors without certs”
1. Click a vendor chip to add a cert, fill in state/cert#/dates, mark as sent
1. See the KPI cards update in real-time
1. Export the full database as JSON
1. Upload a W-9 PDF and have it linked to a vendor

Build this from scratch. Do not use any pre-built admin templates. The UI must match the AiPM design system described above.
